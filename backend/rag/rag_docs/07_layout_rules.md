# 资源布局决策规则

## 五种布局类型

### 1. 高适宜综合承载区
- 特征：综合得分排名前 5，需求网络与能源低碳优势均衡
- 布局导向：优先布局
- 功能定位：承接全国级智算、云计算和数据要素流通平台
- 优化策略：加快绿色数据中心集群建设，强化跨区域算力调度和绿色电力直供
- 风险提示：防止算力设施重复建设与高耗能扩张

### 2. 需求网络驱动区
- 特征：数字产业需求强，网络基础好，但能源或低碳存在短板
- 布局导向：稳步扩容
- 功能定位：依托数字产业、网络基础和市场需求扩大算力供给
- 优化策略：推动边缘算力、行业云和城市级算力节点建设，增强能源侧约束治理
- 风险提示：避免需求扩张快于绿色能源保障能力

### 3. 能源低碳优势承接区
- 特征：绿色能源丰富，低碳优势明显，但需求和网络基础较弱
- 布局导向：跨区承接
- 功能定位：发挥绿色能源和资源承载优势，承接东部算力外溢
- 优化策略：加强网络通道、运维能力和产业应用导入，发展绿色低碳数据中心
- 风险提示：避免形成"有电无算、有算无产"的单一承接模式

### 4. 综合潜力提升区
- 特征：各维度中等偏下，但部分指标有增长潜力
- 布局导向：重点培育
- 功能定位：从单项优势突破，逐步提升综合承载能力
- 优化策略：聚焦短板维度精准施策，培育特色算力应用场景
- 风险提示：避免多点出击导致资源分散

### 5. 约束控制区
- 特征：多项约束指标突出，承载能力受限
- 布局导向：存量优化
- 功能定位：以节能改造和存量优化为主，控制新增高耗能算力设施
- 优化策略：推动存量数据中心节能改造，发展低功耗边缘算力
- 风险提示：高耗能扩张可能加剧碳排放压力

## LPA 到布局映射规则
- 高位领先型 → 高适宜综合承载区（5/5）
- 优势支撑型 → 需求网络驱动区（2/4）或能源低碳优势承接区（2/4）
- 中位追赶型 → 需求网络驱动区（5/9）或能源低碳优势承接区（3/9）或综合潜力提升区（1/9）
- 基础培育型 → 能源低碳优势承接区（3/13）或综合潜力提升区（8/13）或约束控制区（2/13）
