# Claude Code 修改任务：绿色算力决策平台网页展示优化

请基于当前项目代码，对网页前端展示部分进行优化。重点是图表展示、初始状态、交互效果和页面观感升级。请优先修改前端页面与图表组件，不要随意改动后端接口、数据库结构、核心计算逻辑和已有数据字段。

## 一、总体修改要求

1. 保持当前系统整体风格不变：深色科技风、蓝绿色主色调、卡片式布局、数据驾驶舱风格。
2. 不要修改 `.next/`、`standalone/`、`dist/`、`build/` 等打包产物文件，只修改源码文件。
3. 优先复用项目已有图表库和组件。如果当前项目已使用 ECharts、Recharts、AntV 或其他图表库，请沿用现有图表库，不要重复引入多个大型图表依赖。
4. 所有图表都要有合理的标题、图例、tooltip、数据标签或辅助说明，避免页面显得空或者像默认模板。
5. 图表颜色要与当前平台主题统一，建议使用蓝、青、绿、紫、橙等科技感渐变色，不要使用过于鲜艳刺眼的颜色。
6. 图表加载时增加适当的动画效果，例如渐进加载、平滑过渡、hover 高亮、tooltip 动态展示等。
7. 所有页面都要处理空数据状态，但不要让核心页面初始状态空白；需要默认展示上海相关数据。
8. 修改完成后，请确保 `npm run build` 可以通过，并说明修改了哪些文件、每个页面完成了哪些调整。

## 四、区域差异页面修改

要求：

1. 四幅图分别使用不同的图表类型。
2. 不要四张图都长得一样。
3. 每张图都要适配当前深色主题。

推荐组合：

- 图 1：折线图或面积折线图，用于展示时间趋势。

- 图 4：雷达图、极坐标图或散点气泡图，用于展示多维差异特征。

请根据当前页面已有数据结构选择最容易稳定实现的方案。如果某种图表缺少必要数据，可以选择功能等价但更适合当前数据的图表。

## 十一、错误处理与体验优化

1. 页面不要因为某个图表数据缺失而整体崩溃。
2. 图表数据为空时，显示友好的空状态提示，例如“暂无该省份数据，请切换其他省份”。
3. 不要把所有错误都显示为“服务暂时不可用”，应尽量展示具体错误或友好提示。
4. 智能问答中，如果用户提出无关问题，不要抛出 500 错误，应该正常返回 out_of_scope 提示，例如：
   “当前系统主要支持绿色算力承载能力评估、排名、障碍诊断、空间格局和布局建议等问题。”
5. 保证前端交互不会因为接口返回空数组、null 或 undefined 报错。
